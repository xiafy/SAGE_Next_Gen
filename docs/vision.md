## 一、我们在做什么

**SAGE 是一个面向全球旅行者的餐饮智能体（Dining Agent）。**

不是菜单翻译工具，不是点餐 App，不是评分导览。

SAGE 是一个陪你在陌生餐桌前做决定的 AI 伙伴——你不认识菜单上的字，不了解这里的饮食文化，不知道哪道菜适合你此刻的状态。SAGE 感知你所在的场景，理解你当下的需求，帮你在 30 秒内做出一个自己满意的决定。

**核心场景**："陌生"是产品存在的前提，不是边缘场景。
- 操作性定义：**菜单内容超过 30% 是用户母语以外的语言**，即为目标场景。

---

## 二、目标用户

### 最终目标
全球旅行者：任何国家的人，去全球任意地方旅行时的用餐场景。

### MVP 范围（刻意收窄）
**中国旅行者** + **英语系欧美旅行者**

**选择理由**：
1. 控制项目初期复杂度，尽快投入市场验证
2. 这两类用户潜在基数足够大
3. Mr. Xia 相对了解这两个群体，触及渠道可及

**两类用户的核心痛点对比**：

| | 中国旅行者 | 英语系欧美旅行者 |
|--|-----------|----------------|
| 最常遇到的陌生菜单 | 欧美/东南亚/日韩菜单 | 亚洲语言菜单（中/日/韩/泰）|
| 语言障碍 | 英文/日文/泰文等 | 中文/日文/韩文/泰文等 |
| 文化障碍 | 西餐礼仪/点餐规则/小费 | 食材认知/饮食禁忌/菜系背景 |
| 共同需求 | 理解菜单 + 做出符合自己口味的点餐决策 ||

## 三、核心场景定义

**"陌生"不是偶发状态，是旅行者的常态。**

典型场景：
- 🇯🇵 中国游客在东京居酒屋，菜单全是日文手写
- 🇹🇭 美国旅行者在曼谷夜市，指着图片不知道点什么
- 🇫🇷 中国商旅在巴黎，看不懂法语酒单，不敢开口问
- 🇰🇷 美国留学生在首尔，每次点餐都靠比手画脚

**不在目标范围的场景**（MVP 阶段）：
- 用户在熟悉语言的餐厅（菜单母语 ≥ 70%）
- 外卖/线上点餐（需要完全不同的产品形态）
- 餐厅推荐/选择（用户已在餐厅，不是找餐厅）

## 四、核心哲学

### 4.1 Goal > Process
不设计流程图，只设终极目标：**帮用户在这顿饭上做出最好的决定**。

流程由 AI 在当下动态生成，不写死任何业务逻辑树。

### 4.2 Scenario-Free（无预设场景）
不给用户贴标签（急行军/探索者/避雷针）。

标签 = 牢笼。同一个人，今天是赶时间的出差者，明天是和朋友探索夜市的旅行者，后天是带父母吃本地特色的孝顺子女。把全息 Context 传给 AI，让它自己推理，不替它做判断。

### 4.3 多做感知层，少做判断层
感知层 = 收集并传递上下文（菜单内容、GPS、时间、天气、历史偏好）

判断层 = 基于规则做决策（❌ 不做）

信任大模型的涌现能力，不用 if-else 替代推理。

## 五、4+1 维感知体系

| 维度 | 数据来源 | 旅行场景中的信号含义 |
|------|---------|-------------------|
| 👁 **视觉** | 菜单照片 | 菜单语言→目标客群；设计风格→消费层级；菜品种类→饮食文化背景 |
| 📍 **空间** | GPS 坐标 | 旅游区→游客场景需解释文化；机场→赶时间要效率；市区夜市→探索心态 |
| ⏰ **时间** | 系统时间 | 早餐/午餐/晚餐/夜宵对推荐逻辑完全不同 |
| ⛅️ **环境** | 天气 API | 35°C 热带天气→推冷饮冰食；阴冷→推热汤暖胃 |
| 📚 **历史记忆** | 本地存储（MVP）→ 云端（正式版）| 点餐记录→隐性偏好推断；回避食材→过敏/不耐受信号；高频选择→口味倾向 |

**"菜单即场景"洞察**（产品核心认知）：
- 快餐一页纸 = 效率场景，别废话，直接推荐
- 皮质精装酒单 = 宴请/商务场景，讲品位和故事
- 三语图片菜单 = 游客聚集场景，重文化解释
- 手写黑板 = 今日特供，强调新鲜食材

## 七、核心用户价值

**一句话**：在陌生的餐桌前，帮旅行者在 30 秒内做出一个自己满意的点餐决定。

**具体价值点**：
1. **消除语言障碍** — 看不懂的菜名、食材、点餐规则，AI 全部解读
2. **消除文化焦虑** — 不懂当地饮食文化也能自信点菜（小费规则、辣度标准、食物禁忌）
3. **匹配当下状态** — 不是给你所有好菜，是给你**现在这个场景**最合适的选择
4. **降低决策成本** — 从"纠结"到"确定"，≤ 3 轮对话
5. **记忆进化** — 每次用餐后记录，SAGE 越来越了解你的偏好

## 八、商业模式

**模式**：订阅制（Freemium）

| 层级 | 内容 | 价格 |
|------|------|------|
| 免费试用 | 前 X 次使用完全免费（X 待定，需市场验证）| $0 |
| 订阅版 | 无限次使用 + 历史记忆 + 高级功能 | 按周/月付费，具体价格待调研 |

**定价原则**：由市场和 AI API 成本共同决定，需持续调研和迭代验证。不做一次性定价决策。

**后续需要调研**：
- 同类旅行工具的订阅定价区间
- 目标用户对"餐饮 AI 工具"的付费意愿
- AI API 边际成本（每次菜单识别 + 对话的成本）

## 九、竞争格局

### 直接竞品：无
目前没有以"旅行者餐饮智能体"为定位的垂直产品。

### 主要威胁：通用 AI App 的"够用就好"

| 竞品 | 对我们的威胁 | 我们的差异化 |
|------|------------|------------|
| ChatGPT / Claude | 可以拍照问菜单，无需专门 App | 无场景感知，无历史记忆，操作步骤多，旅行场景体验差 |
| 豆包 | 明确布局出境游菜单翻译，中国用户有黏性 | 只做翻译，无推荐、无记忆、无文化解读 |
| Google Lens | 菜单文字翻译 | 只做翻译，完全无推荐能力 |
| Menuthis AI | AI 生成菜品图片 | 重展示不重决策，幻觉风险高 |

**核心差异化**：SAGE 是**决策工具**，不是翻译工具。差别在于从"我知道这是什么"到"我知道我该点什么"。

**关键验证数据**：Luxury Escapes 对话式 AI 实现官网转化率 3x——验证了 Conversation-First 在高决策压力场景的优势。旅行者面对陌生菜单时的决策压力 = 典型高压决策场景。

## 十、MVP 验收标准（Beta 阶段）

**测试对象**：Mr. Xia + 10-20 位朋友（中国旅行者、美国华人/留学生、少量美国人）

**"陌生餐厅"操作定义**：菜单内容超过 30% 是用户母语以外的语言。

**三层验收指标**：

| 层次 | 指标定义 | 阈值 |
|------|---------|------|
| **激活** | 在陌生餐厅实际打开并完成菜单扫描 | ≥ 80% 的用户完成过 ≥1 次 |
| **价值传递** | 用户通过与 App 交互并最终得到推荐 | ≥ 60% 的使用次数 |
| **复购意愿** | 表示下次陌生餐厅就餐时还会使用 | ≥ 50% 的用户 |

**测试周期**：Beta 用户获得访问后，2 周内收集数据（旅行频率不同，给足时间）

## 十二、迭代哲学

**Hyper-Agile**：
- 规划单位：小时 / 天
- 最长跨度：周
- 不做月度路线图，不做季度 OKR

**自主工作协议**（Agent 工作准则）：
- Auto-Chaining：做完 A 自动启动 B
- Proactive Push：主动推送里程碑，不等询问
- Only Stop When Blocked：完成或遇真正障碍才反馈
