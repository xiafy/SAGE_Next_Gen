## T2 — 竞品深度分析

### 目标
输出一份可指导产品差异化决策的竞品分析文档，不是泛泛罗列，而是聚焦"旅行者在陌生餐厅场景下"各竞品的实际表现和短板。

### 研究方法
1. **Web 搜索**：查询各竞品最新功能、用户评价、旅行场景下的实际使用反馈
2. **场景模拟**：对每个竞品模拟"中国旅行者在泰国餐厅"和"美国旅行者在日本餐厅"两个核心场景，分析用户完成点餐决策需要几步
3. **差距分析**：找出每个竞品在 SAGE 核心场景下最明显的能力缺口

### 研究对象
- ChatGPT（含 Vision）
- 豆包（Doubao）
- Google Lens + Google Translate
- Menuthis AI
- 其他新兴竞品（搜索发现）

### 文档结构
1. 竞品概览矩阵（功能对比）
2. 核心场景步骤分析（操作路径对比）
3. 每个竞品的详细分析（优势/劣势/最大威胁点）
4. SAGE 差异化机会点
5. 需要警惕的风险

### 成功标准
- [ ] 覆盖至少 4 个竞品的深度分析
- [ ] 有具体的"操作步骤数"对比数据
- [ ] 明确列出 SAGE 的 3 个以上差异化机会点
- [ ] 有"我们应该警惕什么"的风险提示

### 预计耗时：40 分钟

## T3 — UX 设计原则

### 目标
从已确认的产品哲学（Conversation-First / Scenario-Free / 感知层优先）推导出具体可执行的交互设计原则，让设计师和 Agent 在做任何 UI 决策时有明确的判断依据。

### 内容框架
1. **核心设计哲学**（从 VISION.md 提炼）
2. **对话交互原则**（AI 主动性 / 快捷按钮 / 响应节奏）
3. **移动端旅行场景原则**（单手操作 / 弱网 / 嘈杂环境 / 紧张情绪）
4. **信息密度原则**（旅行者状态下的认知负荷）
5. **错误处理原则**（优雅降级 / 不让用户感到愚蠢）
6. **反模式清单**（明确禁止的设计决策）

### 成功标准
- [ ] 每条原则有"这样做 ✅ / 不这样做 ❌"的对比示例
- [ ] 覆盖 Path A 核心流程的所有关键交互节点
- [ ] 新 Agent 读完后能独立判断一个 UI 设计是否符合原则
