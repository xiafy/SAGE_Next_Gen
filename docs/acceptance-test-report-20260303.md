## 🔴 P0 严重问题（阻塞上线）

### BUG-001: Explore 菜品数据全部错误
- **位置**: 菜单浏览页（Explore）
- **现象**: 3 个分类下的所有菜品都显示为"泰式煎蛋盖饭 / Rice topped with Thai omelet"，价格全是 70.-，描述全是"菜单识别结果（待补充详情）"
- **原因推测**: 菜单识别返回的 items 数组没有正确映射到 Explore 的渲染数据，可能是 ID 重复或数据覆盖
- **影响**: 用户看到的菜单完全不可用
- **截图**: explore-all-same.png

### BUG-002: Order 将不同菜品合并为同一条目
- **位置**: 点餐单页 + Waiter 模式
- **现象**: "整套加入订单"后，泰式煎蛋 + 猪肉炒河粉 被合并为 "Rice topped with Thai omelet ×2"
- **原因推测**: addToOrder 使用了相同的 ID 或 name 匹配，导致两道不同菜被视为同一道
- **影响**: 给服务员展示的订单完全错误，会导致上错菜
- **截图**: waiter-wrong-order.png, order-merged.png

### BUG-003: MealPlanCard 替换后不更新
- **位置**: Chat 中的 MealPlanCard
- **现象**: 点击替换按钮后，AI 文字回复了替换建议（猪肉盖饭），但 MealPlanCard 仍显示原始方案（泰式煎蛋），没有生成 v2 卡片
- **原因推测**: 替换触发了 Chat 消息，但没有触发新的 JSON 代码块解析生成新卡片
- **影响**: 用户以为换好了，但"整套加入订单"仍然加的是旧方案

### BUG-004: Explore 数量与 Order 耦合
- **位置**: 菜单浏览页
- **现象**: Explore 中每道菜的 +/- 数量显示为 2（来自 Order 数据），应该是独立的或不显示数量
- **影响**: 用户在浏览菜单时看到错误的数量信息

## 🟡 P1 体验问题（建议修复）

### ISSUE-005: Pre-Chat 过敏问题被跳过
- **位置**: Pre-Chat 阶段
- **现象**: AI 问"有没有忌口或过敏"后，没等用户回答就进入了主 Chat（因为菜单识别完成触发了 handoff）
- **影响**: 用户的过敏信息没有在 Pre-Chat 收集，需要在主 Chat 重复说明
- **建议**: handoff 应该等 Pre-Chat 轮次完成，不因识别完成而提前中断

### ISSUE-006: 价格不一致
- **位置**: MealPlanCard vs Order 底部栏
- **现象**: 卡片显示 ฿135，底部显示 THB 140
- **影响**: 用户对价格产生疑惑

### ISSUE-007: 货币符号不统一
- **位置**: 全局
- **现象**: MealPlanCard 用 "฿"，Order 用 "THB"，Explore 用 "70.-"
- **建议**: 统一为一种格式
