## 详细发现

### 发现 1: VL+Enrich "两阶段" vs DEC-068 单阶段合并 — 多文档未同步
- **严重度**: 🔴 矛盾
- **涉及文档**: api-design.md (§1.3, §2.3, §3.1) / architecture.md (§4.2) / prd.md (F03) vs DEC-068 + menuAnalysis.ts
- **具体内容**: DEC-068（2026-03-03）将 VL+Enrich 合并为单次 Gemini 调用（Prompt v9），`analyze.ts` 只保留 Step 1。但 api-design.md 虽在 v3.0 更新了变更说明头部，正文的模型表（§1.3）、降级模式（§2.3）、Worker 处理流（§3.1）仍大量描述两阶段架构。architecture.md §4.2 整节标题仍为"两阶段 Pipeline"。prd.md F03 仍写两个 Step。
- **建议修复**:
  1. api-design.md §1.3 模型表：VL 和 Enrich 合并为一行
  2. api-design.md §3.1 Worker 处理流：移除 Enrich 相关步骤
  3. architecture.md §4.2：重写为"单阶段 Pipeline"
  4. prd.md F03：改为"单次 AI Vision 调用完成 OCR + 语义补全（DEC-068）"

### 发现 2: /api/analyze SSE stage 名称三方不一致
- **严重度**: 🔴 矛盾
- **涉及文档**: api-design.md §2.2 vs architecture.md §3.2 vs shared/types.ts
- **具体内容**:
  - api-design.md: `uploading | preparing | vision_flash | vision_plus_fallback | validating | completed`
  - architecture.md: `uploading | preparing | analyzing | validating | completed`
  - shared/types.ts: 包含 `enrich_error` 但不含 `analyzing`；包含 `vision_flash`/`vision_plus_fallback`
- **建议修复**: 以 shared/types.ts 为权威，统一更新两份文档

### 发现 3-4: PRD/TOOLS.md 模型名称过时
- **严重度**: 🟡 漂移
- **建议修复**: PRD F03 改为 gemini-2.0-flash；TOOLS.md 更新模型列表

### 发现 5: Chat Prompt 输出格式文档与实际不符
- **严重度**: 🟡 漂移
- **具体内容**: api-design.md §4.2 要求纯 JSON 输出，实际 agentChat.ts 为自然语言+末尾 JSON 代码块（DEC-052v2）
- **建议修复**: 重写 §4.2 反映当前格式

- **严重度**: 🟡 漂移
- **具体内容**: api-design.md 的 MealPlan.courses.items 只有 `dishId/name/reason`，缺少 `nameOriginal/price/quantity`（shared/types.ts 和 Prompt 都有）
- **建议修复**: 对齐 shared/types.ts MealPlanItem
