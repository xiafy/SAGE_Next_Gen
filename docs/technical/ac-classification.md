# AC 分类与诚实覆盖率评估

> 日期: 2026-03-09
> 方法论: 每条 AC 按"能证明满足需要什么层级的测试"分类，逐条评估现有测试是否真正证明了 AC

## 分类定义

| 类型 | 含义 | 验证手段 |
|------|------|---------|
| **A** | 逻辑型 — AC 描述数据/状态行为 | 单元测试（reducer / 纯函数） |
| **B** | 组件行为型 — AC 描述 UI 条件渲染/交互 | 组件测试（render + assert DOM） |
| **C** | 集成型 — AC 涉及跨模块协作/网络 | 集成测试（多模块 mock） |
| **D** | AI 质量型 — AC 描述 AI 输出质量 | Prompt 断言 + 输出反模式检测 |
| **E** | 性能/硬件 — 需真机测量 | 真机测试，排除自动化覆盖率 |
