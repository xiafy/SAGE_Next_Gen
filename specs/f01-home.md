# F01 — Home Spec

> 从 PRD F01 提取，Sprint 1 实现

## 概述

Home 是用户进入 SAGE 的第一个页面。目标是让用户打开 App 后 1 秒内看到操作入口，零摩擦开始任务。采用无 Tab Bar 的纯任务流导航架构（DEC-040），根据是否有进行中会话动态展示不同入口。

## 用户故事

- 作为首次用户，我打开 App 就看到「扫描菜单」大按钮，立即知道该做什么。
- 作为回访用户（有未完成会话），我看到「继续上次用餐」和「新的一餐」两个入口，可以快速恢复或重新开始。

## 交互流程

```
[打开 App]
  ├─ GPS 授权请求（立即、静默）
  ├─ 系统语言检测 → UI 语言设置
  ↓
[Home]
  ├─ 右上角：⚙️ 设置（→ Settings 页面）
  ├─ 动态问候语（基于时段：早安/午好/晚好/深夜好）
  │
  ├─ 状态 A（无进行中会话）：
  │    └─ 「📷 扫描菜单」大按钮（视觉焦点）→ Scanner
  │
  └─ 状态 B（有进行中会话，menuData 存在）：
       ├─ 「🍽 继续上次用餐」→ 恢复 AgentChat
       └─ 「🔄 新的一餐」→ 清空会话数据 → Scanner
```

## 数据模型

引用 `shared/types.ts`：
- `Language`：`'zh' | 'en'`，用于 UI 语言显示
- `MenuData`：判断是否有进行中会话（menuData 是否存在于 AppContext）

会话感知逻辑：
```typescript
const hasActiveSession = !!appState.menuData;
```

## 验收标准

- [x] AC1: 首屏可交互 < 2 秒（4G 网络）
- [x] AC2: 主入口在 375px-430px 屏宽下清晰可见，无需滚动
- [x] AC3: 语言根据系统语言自动显示
- [x] AC4: 设置图标可访问，点击进入设置页
- [x] AC5: 有进行中会话时显示「继续上次」和「新的一餐」两个入口
- [x] AC6: 「新的一餐」清空会话数据后进入 Scanner

## 边界情况

| 场景 | 处理 |
|------|------|
| 系统语言非中文/英文 | 默认使用英文 |
| localStorage 数据损坏 | 视为无会话状态，显示「扫描菜单」 |
| 网络完全离线 | Home 页可正常渲染（纯静态），后续功能显示网络提示 |
| 极窄屏幕（< 320px） | 按钮自适应宽度，文案不截断 |
